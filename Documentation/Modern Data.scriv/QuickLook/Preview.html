    • Streaming model

      • Overview
        There are two types of object used to control streaming, streams and processors. Each has a transparent part, which is a structure containing pointers to callback functions, and an opaque part, which is represented as a void pointer. To perform input - deserialization - client code provides a stream, and requests a processor from the system. Output - serialization - is the reverse, with client code providing a processor and requesting a stream from the system.
